  <action>
Create `app/api/solve/route.ts` as a Next.js App Router route handler. This is the core LLM integration route.

**Implementation details:**

1. Import `streamText`, `Output` from `'ai'`; `google` from `'@ai-sdk/google'`; `solveRequestSchema`, `solvedAnswerSchema` from `'@/lib/schemas'`.

2. Export `POST` async function accepting `Request`:
   - Parse body with `await request.json()`
   - Validate with `solveRequestSchema.safeParse(body)` — return 400 JSON `{ error: "Invalid request" }` if validation fails
   - Extract `questions` and `mode` from validated data

3. Build message content parts array:
   - First part: `{ type: 'text', text: prompt }` where prompt is a system-style instruction. Use this prompt template (hardcoded for now — Plan 02-02 will make it mode-aware):
     ```
     You are solving a Brazilian university (UNIP) multiple-choice exam.
     For each question, return the correct answer letter.
     Questions are numbered starting from the array order (0-based index).
     ${questions.length} questions follow.

     ${questions.map((q, i) => `[Question ${i}] ${q.text}\nOptions:\n${q.options.map(o => `${o.letter}) ${o.text}`).join('\n')}${q.imageBase64 ? '\n[This question has an accompanying image]' : ''}`).join('\n\n')}
     ```
   - For each question with `imageBase64` (non-null), push an image part:
     ```typescript
     { type: 'image' as const, image: q.imageBase64, mimeType: 'image/png' as const }
     ```

4. Call `streamText`:
   ```typescript
   const result = streamText({
     model: google('gemini-2.5-flash'),
     output: Output.array({ element: solvedAnswerSchema }),
     messages: [{ role: 'user', content: parts }],
   });
   ```
   Note: do NOT await streamText — it returns synchronously. The result has `.toTextStreamResponse()` for streaming.

5. Return `result.toTextStreamResponse()` — this sends the streaming response with proper headers.

6. Add `export const maxDuration = 60;` at the top level (for Vercel Pro plan timeout extension).

**Security:** The route runs server-side only. `GOOGLE_GENERATIVE_AI_API_KEY` is read from env by `@ai-sdk/google` automatically — never import or reference it in code. Never return raw env vars in responses.

**Error handling:** Wrap the streamText call in try/catch. On error, return `NextResponse.json({ error: "Failed to solve questions" }, { status: 500 })`. Import `NextResponse` from `'next/server'`.
