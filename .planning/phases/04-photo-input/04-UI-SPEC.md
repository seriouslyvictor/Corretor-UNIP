## Layout Contract

### Tab Toggle

Reuse the exact pattern from the existing mode selector in `page.tsx`:

```
<div className="border border-border rounded-lg p-1 flex gap-1">
  <Button variant={tab === "html" ? "default" : "ghost"} className="flex-1 gap-1">
    <Code size={16} /> HTML
  </Button>
  <Button variant={tab === "photo" ? "default" : "ghost"} className="flex-1 gap-1">
    <Camera size={16} /> Foto
  </Button>
</div>
```

- Container: `border border-border rounded-lg p-1 flex gap-1`
- Active tab: `variant="default"` (filled primary)
- Inactive tab: `variant="ghost"` (not "outline" — ghost blends into the container background)
- Tab labels in Portuguese: "HTML" | "Foto"

### HTML Tab (modified)

Remove the file upload Card entirely. Keep only:
1. The paste Textarea (`Código-fonte HTML` label + `<Textarea>`)
2. The submit button (`Corrigir prova`)

Updated subtitle text: "Cole o HTML da revisão de prova abaixo." (drop "ou carregue o arquivo salvo" — file upload is gone per D-02)

### Photo Scan Tab

Layout (top to bottom, single column, `flex flex-col gap-4`):

1. **Thumbnail grid** — shown only when ≥1 image queued; hidden (not rendered) when queue is empty
2. **"+ Adicionar imagem" button** — always visible; opens file/camera picker
3. **"Extrair Questões →" CTA button** — always rendered; `disabled` until ≥1 image queued

#### Thumbnail Grid

```
<div className="grid grid-cols-2 gap-2 sm:grid-cols-3">
  {images.map(...)}
</div>
```

- Grid columns: 2 columns on mobile (default), 3 columns at `sm` breakpoint (640px+)
- Each thumbnail: `Card` with `CardContent` at `p-0 relative overflow-hidden` and fixed `aspect-square` size
- Image preview: `<img>` with `object-cover w-full h-full` inside the card
- Remove button: absolute-positioned top-right corner, `absolute top-1 right-1`, `size-6` icon, 44px minimum tap target via `p-1` wrapper
- Remove button style: `bg-background/80 rounded-full` — semi-transparent, not a full button variant

#### Empty State (no images queued)

No illustration. Render only the "+ Adicionar imagem" button and the disabled "Extrair Questões →" button. Zero friction — immediate action.

No empty state text, no placeholder card. The presence of two buttons is sufficient affordance.

#### "+ Adicionar imagem" button

```
<Button variant="outline" className="w-full gap-2" onClick={() => fileInputRef.current?.click()}>
  <Plus size={16} /> Adicionar imagem
</Button>
```

- Full width
- `variant="outline"` (secondary visual weight — not the primary action)
- Hidden `<input type="file" accept="image/*" capture="environment" multiple ref={fileInputRef} />`

#### "Extrair Questões →" CTA

```
<Button
  variant="default"
  size="lg"
  className="w-full gap-2"
  disabled={images.length === 0}
  onClick={handleExtract}
>
  Extrair Questões <ArrowRight size={16} />
</Button>
```

- Full width, `size="lg"` — matches existing "Corrigir prova" CTA
- `disabled` when `images.length === 0`; enabled (primary filled) when ≥1 image queued
- Icon after text (arrow-right pointing forward = proceed)

## Accessibility Contract

- `<input type="file">` is hidden; triggered by button click only. Button must be keyboard-focusable (it is, as a `<Button>`).
- ✕ remove buttons must have `aria-label="Remover imagem"` (image has no text label).
- "Extrair Questões →" must have `aria-disabled="true"` when `disabled` prop is set (shadcn Button handles this).
- Tab toggle buttons: add `aria-pressed={tab === "html"}` and `aria-pressed={tab === "photo"}` — matches existing mode selector pattern in `page.tsx`.
- Error alerts: `role="alert"` — matches existing error pattern in `page.tsx`.
