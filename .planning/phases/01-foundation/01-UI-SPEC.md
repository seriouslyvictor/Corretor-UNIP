# Phase 1 — UI Design Contract

> Visual and interaction contract for the HTML input page of Corretor UNIP.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Screens in Scope

This contract covers two screens delivered in Phase 1:

1. **Input page** (`app/page.tsx`) — landing screen where the user provides a UNIP test HTML file and selects a solving mode.
2. **Loading state** — full-page overlay or replacement state shown after submit while Gemini streams answers.

The gabarito results page is Phase 3 and is explicitly out of scope here.

## Spacing Scale

Standard 8-point scale. All layout spacing must use Tailwind v4 utility classes mapped to these values.

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps, inline tight spacing |
| sm | 8px | `gap-2`, `p-2` | Label-to-input spacing, within-card tight grouping |
| md | 16px | `gap-4`, `p-4` | Default element spacing, card internal padding |
| lg | 24px | `gap-6`, `p-6` | Card padding (larger), between major form sections |
| xl | 32px | `gap-8`, `p-8` | Between card and submit button, layout column gaps |
| 2xl | 48px | `gap-12` | Page-level vertical center offset |
| 3xl | 64px | `gap-16` | Not used in Phase 1 |

**Exceptions:**
- File upload drop zone minimum height: 120px (not an 8-point multiple — required for usable drag target)
- Touch/click targets for mode toggle buttons: minimum 44px height (accessibility requirement)
- Paste textarea minimum height: 160px

## Color

All color values reference the CSS custom properties already defined in `app/globals.css`. Dark theme is the default (`.dark` class on `<html>`). Light theme uses `:root` values. Do not hardcode OKLCH values — use semantic token names.

| Role | CSS Token | Dark Mode Value | Usage |
|------|-----------|-----------------|-------|
| Dominant (60%) | `bg-background` | `oklch(0.148 0.004 228.8)` — near-black blue | Page background |
| Secondary (30%) | `bg-card` | `oklch(0.218 0.008 223.9)` — dark slate | Card surfaces (upload card, paste card), mode selector background |
| Accent (10%) | `bg-primary` / `text-primary` | `oklch(0.437 0.078 188.216)` — teal | Submit button fill, active mode toggle indicator, focus rings |
| Destructive | `text-destructive` / `border-destructive` | `oklch(0.704 0.191 22.216)` — warm red | Parse error message text and border only |
| Muted text | `text-muted-foreground` | `oklch(0.723 0.014 214.4)` — grey-blue | Placeholder text, helper copy, instruction text |
| Border | `border-border` | `oklch(1 0 0 / 10%)` — translucent white | Card borders, separator, input borders |

**Accent reserved for:** Submit button background, the active/selected state of the No BS / Verbose mode toggle, and focus ring outlines (`ring-primary`). Accent is NOT used on card backgrounds, section headings, or decorative elements.

**OR separator:** Use `text-muted-foreground` + `border-border` via the shadcn `Separator` component. Not styled with accent.

Primary visual anchor: the "Corrigir prova" submit button at the bottom of the input form.

## Interaction Contract

| Trigger | Behavior |
|---------|----------|
| Click anywhere in upload Card | Opens native file picker (`input[type=file].click()`) |
| Drag file over upload Card | Card border changes to `border-primary` (accent); background tint `bg-primary/5` |
| Drop valid `.html` file | File name shown as `<p className="text-sm text-muted-foreground">` replacing upload copy; Card border returns to default |
| Drop invalid file type | Inline error below card: "Apenas arquivos .html são aceitos." (`text-destructive`) |
| User pastes into Textarea | No immediate validation; validated only on submit |
| Click "No BS" button | Button becomes `variant="default"` (accent fill); Verbose returns to `variant="outline"` |
| Click "Verbose" button | Button becomes `variant="default"` (accent fill); No BS returns to `variant="outline"` |
| Default mode on page load | "No BS" is pre-selected (accent fill) |
| Click "Corrigir prova" with no input | Parse error appears: "Nenhuma questão encontrada. Verifique se copiou o HTML correto." |
| Click "Corrigir prova" with valid input | Input page replaced by Loading state immediately (no delay) |
| Parse error from lib/parser.ts (0 questions) | Same inline error as above |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| File input keyboard access | Upload Card has `tabIndex={0}` and `onKeyDown` (Enter/Space triggers click) |
| Mode buttons | Standard `<button>` elements — keyboard accessible by default; `aria-pressed` reflects active state |
| Submit button disabled state | `disabled` attribute when no input present AND after submit while loading |
| Error association | Parse error `<p>` has `role="alert"` so screen readers announce it immediately |
| Drag-drop not exclusive | Drag-drop is progressive enhancement; click-to-upload is always the primary path |
| Focus ring | Inherited from shadcn Button/Input: `focus-visible:ring-3 focus-visible:ring-ring/30` |
